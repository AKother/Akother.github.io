第二章 简单动态字符串

简单动态字符串 - Simple Dynamic String

结构

struct sdshdr{
    int len; // 不算空字符串
    int free;
    char buf[];
}
  • 以空字符串结尾（SDS函数自动完成添加和删除）–> 好处是可以重用C语言中的字符串函数库

SDS与C字符串的区别

  • 常量获取字符串长度，因为保存了len，C只能遍历
  • 杜绝缓冲区溢出（不管是直接分配还是拼接(sdscat))，因为保存了free，而C可能会假定内存够，实则溢出
  • 减少修改字符串时带来的内存重分配技术：C在截断或拼接时都要管理内存，否则可能会有内存溢出或者内存泄漏，而Redis实现如下：
    • 空间预分配：要对SDS空间扩展时，同时分配需要空间和额外空间
      • 修改后len<1MB: len = free
      • 修改后len>1MB: free = 1MB
    • 惰性空间：String中移除字符后不会立刻释放空间，由free记录（可能以后用到）（源码：用户再次调用该字符串，会调用释放空间的API）
  • **二进制安全：**C字符串有编码限制（不能保存图像、音频等二进制数据），但Redis用的buf[]则可以
  • Redis有需要时也可以重用C语言函数库（感觉这个不算区别）

第四章 字典

字典，又称为符号表（symbol table）、关联数组（associative array）或映射（map），是一种用于保存键值对（key-value pair）的抽象数据结构。

Redis的数据库就是使用字典来作为底层实现的，对数据库的增、删、查、改操作也是构建在对字典的操作之上的。

解决键冲突

链地址法解决

rehash

当哈希表保存的键值对数量太多或者太少时，程序需要对哈希表的大小进行相应的扩展或者收缩。

步骤

  • 为ht[1]分配空间（根据使用空间分配）

    • 如果是扩展操作，ht[1]大小=第一个大于等于ht[0].used*2的2^n

    • 如果是收缩操作，ht[1]大小=第一个大于等于ht[0].used的2^n

      (这里描述的有点难受，实际上就是，如果扩3，要变为4；如果要扩9，要变为16，只能是2的整数倍)

  • 将ht[0]的所有键重新计算索引值，并将键值对一起放到ht[1]指定位置

  • 迁移完毕以后释放ht[0]，另ht[0]=ht[1]（新），再新建一个ht[1]

哈希表的扩展与收缩时机

扩展
  • 哈希表负载因子>=1，服务器没有执行BGSAVE或BGREWRITEAOF
  • 哈希表负载因子>=5，服务器正在执行BGSAVE或BGREWRITEAOF

也就是说只要扩展因子>=1就有可能实现哈希表扩展或收缩

收缩
  • 哈希表负载因子<=0.1，自动开始收缩

渐进性rehash

这也是实际上的用法，渐进式rehash中字典会同时用ht[0]和ht[1]（可能先去ht[0]找，之后再进ht[1]）

为了避免rehash对服务器性能造成影响，服务器不是一次性将ht[0]里面的所有键值对全部rehash到ht[1]，而是分多次、渐进式地将ht[0]里面的键值对慢慢地rehash到ht[1]。

  • 为ht[1]表分配空间
  • 在字典中维护索引计数器rehashidx，将值设为0时表示rehash开始
  • 每次增删改查，都会顺带将ht[0]在rehashidx索引上的所有键值对rehash到ht[1]，并将rehashidx+1
  • 某个时间点上ht[0]的所有值都会被rehash到ht[1]，此时令rehashidx=-1表示rehash操作完成

注意：

  • 新增key-value全都会被保存到ht[1]，这个步骤保证ht[0]只增不减，最终变为空表

第五章 跳跃表

跳跃表（skiplist）是一种有序数据结构，它通过在每个节点中维持多个指向其他节点的指针，从而达到快速访问节点的目的。平均O(logN)、最坏O(N)

很多程序用跳跃表来代替平衡树，因为性能差不多但跳表实现更简单

  • 当有序集合数量多，或有值是比较长的字符串时，跳表就被用作有序集合键的底层实现
  • 集群节点用作内部结构

// 跳跃表
typedef struct zskiplist{
    structz skiplistNode *header, *tail; // 表头和表尾
    unsigned long length; // 节点数量
    int level; // 最大层数
}


// 跳表节点
typedef struct zskiplistNode{
    // 层
    struct zskiplistLevel{
        struct zskiplistNode* forward;// 前进指针
        unsigned int span; // 跨度
    }level[];
    
    struct zskiplistNode* backward; // 后退指针
    double score; // 分值｜按分值从小到大排
    robj *obj; // 成员对象
}zskiplistNode;
  • level数组可以包含多个元素，可以通过层来加快访问其他节点的速度，层数越多，访问效率就越高
  • 创建新跳表节点时，随机生成1～32之间的值作为level数组的大小（层的高度）
  • 遍历只需要利用前进指针
  • Span说明两个节点之间的距离，计算rank：查找某个节点过程中，将沿途所有层的跨度累计起来，结果就是目标节点在跳表中的排位
  • 同个跳表中，成员唯一但是score可以相同（越小越靠近表头）

第六章 整数集合

整数集合（intset）是集合键的底层实现之一，当一个集合只包含整数值元素，并且这个集合的元素数量不多时，Redis就会使用整数集合作为集合键的底层实现。

结构

typedef struct intset {
    uint32_t encoding; 	// 编码方式
    uint32_t length;	// 集合包含元素的数量
    int8_t contents[];  // 保存元素的数组｜这里的int8_t可以不看
}
  • 保存类型为int16_t / int32_t / int64_t
  • 每个元素按从小到大排列（为了二分插入时判断重复项性质），不会出现重复元素
  • 声明是int8_t，实际上存储的类型取决于encoding

升级

时间复杂度O(N)

  • 根据新元素类型，扩展空间大小，为新元素分配空间
  • 现在有的数组全部都转换为和新元素一样的类型
  • 将新元素放进去

PS：不支持降级，升了就是升了

升级的好处

  • 提高灵活性（不会把两种类型的值放在一个数组）
  • 节约内存（可以保存三种类型的值，又可以保证升级只有在需要时进行，节约内存）
